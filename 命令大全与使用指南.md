## 一、电子宠物系统

### `/buddy` 或 `/buddy show`
显示当前宠物状态卡片（名字、物种、等级、稀有度、五维属性）。

**示例：**
```
/buddy
/buddy show
```

### `/buddy hatch`
重新孵化一只新宠物，随机决定物种、稀有度和闪光状态。
**警告：这会覆盖现有宠物存档！**

---

### `/buddy pet`
抚摸宠物，增加 petCount 和少量 XP。

---

### `/buddy train`
训练宠物，增加较多 XP 并随机提升一项属性。

---

### `/buddy card`
查看完整属性卡，包括成就列表和经验进度。

---

### `/buddy name <名字>`
给宠物起名字。

**示例：**
```
/buddy name 小火龙
```

---

### `/buddy open`
提示在浏览器中打开 `buddy-pet.html`。

---

### `/run-buddy [--screenshot-only] [--interact]`
运行自动化驱动脚本，在 headless 浏览器中与宠物互动并截图。

| 参数 | 说明 |
|------|------|
| 无参数 | 点击抚摸+喂食，截图对比前后状态 |
| `--screenshot-only` | 只加载页面并截图，不点击任何按钮 |
| `--interact` | 额外点击训练+进化+画布，完整交互测试 |

**示例：**
```
/run-buddy
/run-buddy --interact
/run-buddy --screenshot-only
```

**输出文件：**
- `screenshots/01-initial.png` — 页面初始状态
- `screenshots/02-after-interact.png` — 交互后状态

**底层命令：**
```bash
node scripts/buddy-driver.mjs [--screenshot-only] [--interact]
```

---

### `/screenshot-buddy`
快速给宠物页面截一张图，等同于 `/run-buddy --screenshot-only`。

---

### `/ascii-pet`
手动查看 ASCII 宠物动画的当前帧，不递增索引。适合想反复欣赏同一帧时使用。

**示例：**
```
/ascii-pet
```

**输出示例：**
```
    /\_/\
   ( o.o )
    > ^ <
   /|   |\
  (_|   |_)

端坐 — 万子轩，你又完成了一个小任务！
```

**说明：**
- 每次对话（或任务）结束时，系统会自动输出下一帧动画
- 帧序列循环：端坐 → 抬爪 → 舔毛 → 眯眼笑 → 摇尾巴 → 伸懒腰 → 歪头杀 → 踩奶 → 端坐...
- 帧数据存储在 `.claude/ascii-pet-frames.json`，索引在 `.claude/ascii-pet-index.txt`

## 二、项目管理

### `/status`
查看项目全景状态，包括：
- 各类文件数量统计（HTML / MD / JSON / TXT）
- notes/ 目录笔记数量

**示例：**
```
/status
```

**输出示例：**
```

━━━━━━━━━━━━━━━━━━━━

```

---

### `/backup [备注]`
将项目打包为带时间戳的 zip 文件，保存在 `backups/` 目录。

**示例：**
```
/backup
/backup 完成机器人调研报告
```

**输出文件格式：**
```
backups/backup-20260527-1430.zip
backups/backup-20260527-1430-完成机器人调研报告.zip
```

**包含内容：**
- 所有 `.html`、`.md`、`.json`、`.txt` 文件
- `notes/` 目录
- `scripts/` 目录

**不包含：**
- `node_modules/`
- `.claude/`
- `backups/`（避免递归）

**底层命令：**
```bash
node scripts/backup.mjs [备注]
```

---

### `/commit [提交信息]`
自动 git 提交当前所有更改。

**行为：**
1. 检查是否为 git 仓库（有 `.git` 文件夹）
2. 若不是，提示运行 `git init`
3. 若是，自动 `git add -A`
4. 若提供了提交信息，直接使用；否则 AI 根据会话内容生成
5. 执行 `git commit`

**示例：**
```
/commit
/commit 添加备份和截图脚本
```

---

### `/pr <标题>`
创建 GitHub Pull Request。

**前置条件：**
- 项目是 git 仓库
- 已配置 GitHub MCP Token
- 当前分支已 push 到 remote

**示例：**
```
/pr 添加新工具生成器
```

**若条件不满足，会输出缺失项清单：**
```
❌ 无法创建 PR：
   1. 当前目录不是 git 仓库 → 运行 git init
   2. 未配置 GitHub Token → 在 Claude Code 设置中添加
   3. 当前分支未 push → 运行 git push -u origin main
```

## 七、常见问题

**Q: `/run-buddy` 报错 `playwright module not found`**
A: 在项目根目录运行 `npm install playwright`。不需要 `npm init`（目录名含中文会导致 init 失败，直接 install 即可）。

**Q: `/backup` 生成的 zip 在哪里？**
A: `backups/backup-YYYYMMDD-HHmm.zip`，位于项目根目录的 `backups/` 文件夹下。

**Q: `/commit` 提示不是 git 仓库**
A: 先运行 `git init` 初始化仓库，然后再用 `/commit`。

**Q: `/new-tool` 创建的文件名是什么？**
A: 会自动将中文转为拼音或保留中文，空格转为连字符。例如 `/new-tool 密码生成器` 生成 `密码生成器.html`。

**Q: 为什么有些 / 命令执行需要很长时间？**
A: `/run-buddy` 需要启动浏览器并截图，`/web-search` 需要联网查询，`/today` 需要拉取新闻。这些都是正常耗时。
